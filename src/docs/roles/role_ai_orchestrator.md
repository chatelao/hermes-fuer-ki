# Role: AI Orchestrator (Operational)

**ID**: `role_ai_orchestrator`

## Description
Handles day-to-day workflow management and observability.

## Accountability
Day-to-day workflow management, progress tracking, deliverable linking, and documentation maintenance.

## Human Interaction
Provides real-time transparency and bottleneck detection to human handlers.

## Key Responsibilities
- Managing daily workflow and task orchestration
- Tracking progress and resource utilization
- Maintaining deliverable linking and traceability
- Automating status reporting and documentation
- Identifying and flagging bottlenecks proactively

## Interaction Guide
The AI Orchestrator ensures machine-led operational efficiency. It acts as the primary source of truth for project status and provides the data required for human decision-making.

### Cross-Role Interactions
- **With Handler of the Start (Strategic)**: Provides strategic overview, health feeds, and closure evidence.
- **With Handler of Exceptions (Tactical)**: Alerts on tolerance deviations and provides data for exception planning.
- **With Delivery Specialist (Technical)**: Coordinates daily tasks, ensures standard compliance, and maintains the traceability ledger.
